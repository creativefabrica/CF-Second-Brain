How to read this dashboard

Metric Cards (top row)

Snapshot of the latest vault state. Each card shows the current value and a delta vs the previous snapshot.

  • Total entries — everything in the vault. Growth means you're using it. Flat means the habit isn't sticking.
  • Decisions — choices with rationale. This is the highest-value content. If this stays at 0 while tasks grow, you're using the vault as a todo list instead of a knowledge base.
  • Tasks — things to do. Compare with Stale count — if tasks grow but stale also grows, you're capturing but not reviewing.
  • Meetings — post-meeting dumps. One meeting save auto-extracts decisions + tasks, so this is the most efficient content type.
  • Stale (>14d) — open tasks untouched for 2+ weeks. Green (down) = you're resolving things. Red (up) = the vault is becoming a graveyard.
  • Overdue — tasks past their due date. Should always be 0. If it grows, either do the task, extend the date, or cancel it.

Distribution Over Time (line chart)

Shows how each content type grows across snapshots. What to look for:

  • Healthy: multiple lines growing — you're capturing decisions, tasks, and meetings.
  • Unhealthy: only the purple (tasks) line grows — the vault duplicates your issue tracker instead of complementing it.
  • Goal: the blue (decisions) line should grow proportionally. Decisions are the hardest to reconstruct later and the most valuable to have recorded.

Task Health Over Time (stacked bar)

Each bar shows the full breakdown of task statuses at that snapshot. What to look for:

  • Healthy: grey (open) shrinks over time as items move to in-progress or get completed. The bar height stays stable or shrinks.
  • Unhealthy: the bar only grows taller — you keep adding tasks but never resolve them. Red (overdue/stale) segments appearing means decay.
  • Blocked growing: you have dependencies on other people. Check waiting_on fields and follow up.

Type Ratio (doughnut)

Current distribution as a percentage. Targets:

  • Tasks <50% of total — otherwise you're just building another inbox.
  • Decisions >20% — this is where the real value lives.
  • Meetings present — if 0, you're not capturing post-meeting context (the highest-leverage content type).
  • People + Projects >0 — means the knowledge graph has nodes to connect things through.

Growth Rate (bar chart)

Net new entries between snapshots. What to look for:

  • Steady bars: the sb habit is sticking.
  • Bars trending to 0: you stopped using the vault. Ask why — too much friction? Wrong content types?
  • Spikes: usually a meeting dump that created many entries at once. Normal and healthy.

Requires 2+ snapshots to display.

Snapshot Comparison (table)

Raw numbers side by side for every snapshot. Use this to compare Week 1 vs Week 3 at a glance. The divider separates distribution metrics (top) from task health metrics (bottom).

Recent Entries

The entries from the latest snapshot. Check the type badges — if something looks miscategorized (e.g., a research note labeled as "task"), that's a signal to adjust how you phrase sb inputs.

The core question this dashboard answers

"Am I building a knowledge graph, or just another inbox?"

A knowledge graph has: decisions with rationale, meetings with extracted context, people as connection points, projects as hubs. An inbox has: a growing list of tasks you'll never look at again.
